## About Me

**Backend engineer** and **network security architect** with **11+ years of experience** — I build production systems that are fast, maintainable, and hardened against failure.

I work across **software architecture**, **infrastructure**, and **security engineering**: APIs, microservices, data layers, network topology, access control, monitoring, and defense-in-depth — all designed to work together as one system.

I care about **reliability under load**, **security by default**, **clean and readable code**, and **automation that removes manual toil**. When I'm not engineering, I create **digital art** and draw inspiration from film.

## Engineering Philosophy

```text
Build with intent.     → Every service has a clear boundary and responsibility.
Secure by default.   → Trust nothing; validate, segment, monitor, and log.
Design for failure.  → Timeouts, retries, circuit breakers, and graceful degradation.
Measure everything.  → Metrics, traces, and alerts before users report problems.
Keep it operable.    → If it cannot be deployed, debugged, and restored — it is not done.
Automate repetition. → Manual steps are debt; pipelines are assets.
```
